## File

- **Filename:** `yarn.lock`
- **Encoding:** UTF-8, indentation-significant, key/value pairs separated by
  spaces; **not** valid YAML (no `:` after keys, multi-key entries comma-joined).
- **Sibling files:** none required at parse time, but **`package.json` is
  needed to recover workspace structure** ‚Äî yarn 1 lockfile encodes only
  resolved deps, not workspace boundaries.

## Schema sketch

```
# THIS FILE IS AUTOGENERATED ...
# yarn lockfile v1


"foo@^1.0.0":
  version "1.0.3"
  resolved "https://registry.yarnpkg.com/foo/-/foo-1.0.3.tgz#abcdef"
  integrity sha512-...
  dependencies:
    bar "^2.0.0"

"foo@^1.0.5", "foo@~1.0.5":
  version "1.0.7"
  ...
```

Keys = comma-separated dep specs that share the same resolved version.

## Conversion inputs

Workspaces are not encoded in the lockfile. Without `manifests`, only the
root project is recovered, and per-dep classification splits along the
lockfile-derivable / manifest-required axis: only `dep` / `optional`
classification is available (the lockfile records
`optionalDependencies` as a separate inner-block sibling of
`dependencies:`); `dev` and `peer` live only on `package.json` and
collapse to `dep`.

| Operation | Option                           | Required?              | Effect when omitted |
|-----------|----------------------------------|:----------------------:|---------------------|
| Parse     | `manifests['']`                  | required for monorepos | source of truth for the `workspaces` field. Without it, the synthesised root cannot distinguish workspace-member entries (which would normally carry `attrs.workspace = true` on their root-edges) from external `0.0.0-use.local` lookalikes ‚Äî both end up as ordinary `dep` edges from the synthesised root. Member entries themselves remain present as graph nodes; they just lose the workspace-edge marker (per ADR-0019 ¬ßC 473-481) |
| Parse     | `manifests[<workspacePath>]`     | recommended            | recovers full `dep` / `dev` / `optional` / `peer` classification; without it, only `dep` and `optional` are recoverable from the lockfile (yarn 1 emits `optionalDependencies:` as a sibling inner-block), while `dev` and `peer` collapse to `dep` (per ADR-0019 ¬ßC 4-row table) |
| Stringify | `manifests`                      | optional               | re-emit reads only the graph; manifests inform the header comment (currently none) |

## Quirks

- Comments on lines starting with `#` are mandatory (header). Discard on parse,
  re-emit on format.
- Strings are quoted only if they contain special characters; emit must follow
  the same rules byte-for-byte to keep diff hygiene.
- Multi-spec keys (`"foo@^1", "foo@^1.2":`) imply both specs share resolution.
- `dependencies:` block lists **constraints**, not resolved versions ‚Äî
  resolution happens by lookup elsewhere.
- Yarn 1.10+ added `integrity`. Older lockfiles have only `resolved` with a
  `#sha1` fragment.

## Degradation rules

| Feature | Action |
|---------|--------|
| Peer virtualization | **flatten** |
| Patches | **strip** |
| Workspace classification | **strip** (but workspaces themselves are preserved if `manifests` provided) |
