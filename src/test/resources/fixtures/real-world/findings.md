Across all 6 fixtures: full real-world cross-family coverage. Adapter-layer fixes
closed family-wide blockers without introducing new ADRs, RECIPE_* codes, or
LossFeature arms:

1. `src/main/ts/formats/_yarn-berry-core.ts` — `parseSpec` accepts a bare
   `<name>@<version>` half and synthesises `npm:` per ADR-0016 §B (matches the
   existing `entryKeyRangeOf` convention for cross-family inputs).
2. `src/main/ts/recipe/resolution.ts` — `peelYarnBerryLocator` drops the strict
   `expectedName` match (npm-alias collapsed entries carry a different
   `spec[0].name` than the `resolution:` locator), and `parse` threads the locator's
   own name through to `parseInner` for registry-URL derivation.
3. `src/main/ts/formats/yarn-classic.ts` — `deriveResolvedFromCanonical` filters
   non-URL `unknown`-canonical raws (yarn-berry `<n>@patch:…` etc. would otherwise
   leak through as a `resolved` field), and the stringify path dedups
   patched/unpatched siblings of any `<name>@<version>` (the patch loss is already
   attributed via `RECIPE_FEATURE_DROPPED`).
4. `src/main/ts/formats/bun-text.ts` — same patched/unpatched sibling dedup at
   stringify (bun-text has no patch protocol).
5. `src/main/ts/formats/_npm-core.ts` — `deriveInstallPathsForStringify` reorders
   the install-path planner so the root-reachable BFS placement (lines 869-893
   inline `while`) runs BEFORE the lexicographic fallback (lines 906-925). When
   sidecar install-paths are absent (cross-family input), this lets the version
   each root / workspace consumer actually depends on claim `node_modules/<name>`
   first; non-dominant versions fall through to the synthetic capsule path. A
   per-iteration BFS drain inside the fallback loop ensures each orphan's
   transitive dependencies nest under its install path (so `react-dom@18`'s
   `scheduler@0.23` nests inside its capsule and survives npm node-resolution
   reparse). Multi-version is supported по существу — npm's recursive
   `node_modules` algorithm is the canonical hoist plan, the planner now mirrors it.
